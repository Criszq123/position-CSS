Las 4 posiciones CSS

Guía visual interactiva — relative, absolute, fixed, sticky

relative

Se mueve, pero conserva su lugar

El elemento se desplaza visualmente desde su posición original, pero el hueco queda reservado — como si todavía estuviera ahí. Los demás elementos no lo ignoran.

Dato clave: position: relative en un padre es fundamental para que absolute funcione correctamente (lo verás abajo).
absolute

Sale del flujo, se ancla al padre

El elemento desaparece del flujo — los demás actúan como si no existiera. Se posiciona relativo al ancestro más cercano con position: relative. Si no hay ninguno, se ancla al <body>.

Contenedor con position: relative

↖ padre (position: relative)

ABS bottom:10px right:10px

Ajusta bottom y right en vivo

10px
10px
.padre {
  position: relative; /* ancla para el hijo */
}

.hijo {
  position: absolute;
  bottom: 10px;
  right: 10px;
} /* Sale del flujo */
!
Error común: olvidar position: relative en el padre. El elemento sale volando hasta el <body> o el primer ancestro posicionado.
fixed

Pegado a la ventana del navegador

Ignora el scroll completamente. Se posiciona relativo al viewport (la pantalla), no al documento. Siempre visible sin importar cuánto bajes.

Simulación de viewport con scroll
FIXED top:12px right:12px

↑ Aunque el contenido haga scroll, el cuadro verde sigue en top:12px right:12px.

Ajusta top y right en vivo

12px
12px
.botón-flotante {
  position: fixed;
  top: 12px;
  right: 12px;
} /* Relativo al viewport, ignora scroll */
✓
Uso típico: headers fijos, botones de "volver arriba", chats flotantes, barras de navegación persistentes.
sticky

Normal… hasta que llega al borde

Actúa como relative hasta que el scroll lo acerca al umbral definido. Entonces se comporta como fixed — pero solo dentro de su contenedor padre.
